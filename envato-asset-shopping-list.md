## The Problem

Six generic stock images rotate across 33 pages. Lagos gets the same hero as Karachi. The bodyguard page shares imagery with the event security page on list cards. Every repeat screams "template site" to Google and visitors. City-specific imagery is the single biggest visual credibility gap.

## File Naming Convention

When downloading, rename each file before adding to `static/images/`:

**Cities:** `city-{slug}-hero.jpg` and `city-{slug}-street.jpg`
Example: `city-lagos-hero.jpg`, `city-lagos-street.jpg`

**Services:** `service-{slug}-hero.jpg` and `service-{slug}-parallax.jpg`
Example: `service-bodyguard-hire-hero.jpg`, `service-bodyguard-hire-parallax.jpg`

**Risk Assessments:** `assessment-{slug}-hero.jpg` and `assessment-{slug}-parallax.jpg`
Example: `assessment-pre-travel-hero.jpg`, `assessment-pre-travel-parallax.jpg`

**Homepage:** `homepage-{section}.jpg`
Example: `homepage-vetted-operators.jpg`, `homepage-how-it-works.jpg`

**Videos:** `video-{purpose}.mp4`
Example: `video-bodyguard-broll.mp4`, `video-dubai-timelapse.mp4`

## Image Optimisation Notes

- Convert all JPGs to WebP before deploying (Hugo can do this with image processing, or use `cwebp` CLI)
- Target file size: under 200KB per image after compression
- The hero-globe.mp4 is 161MB at 4K. Create a 720p web version (~5-10MB) using FFmpeg before launch:
  ```
  ffmpeg -i hero-globe.mp4 -vf scale=1280:720 -c:v libx264 -crf 28 -preset slow hero-globe-web.mp4
  ```
- Consider creating thumbnail crops (400×300) of city images for the homepage grid cards
